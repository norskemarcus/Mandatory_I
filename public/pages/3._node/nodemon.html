Automatisk restart med Nodemon

Nodemon er et værktøj, der gør udvikling af Node.js-applikationer nemmere og mere effektivt. Med Nodemon behøver du ikke manuelt genstarte din server hver gang, du foretager ændringer i din kode. Nodemon overvåger automatisk dine filer og genstarter serveren, når der sker ændringer, hvilket sparer dig tid og besvær.

Dette værktøj er især nyttigt under udviklingsprocessen, da det giver dig mulighed for hurtigt at se resultaterne af dine ændringer uden at afbryde arbejdsprocessen for at genstarte serveren manuelt.

For at indstallere Nodemon kører man denne kommando i terminalen:

npm install -g nodemon

Når man skal bruge Nodemon, går man til den aktuelle mappe, fx. 02._Assignement> og skriver ind:

nodemon.cmd app.js

For at forenkle hvordan man bruger node og nodemon på forskellige maskiner, kan man gøre dette på en anden måde.

Man kan tilføje en nodemon.json fil der man skriver hvilke extensions den skal lytte til og så i package.json tilføje denne kode under scripts : 


      // I terminalen:
      npm i cross-env

      // I nodemon.json:
      {
        "ext": "html, js, json, css"
      }
      
      // I package.json (efter "test"):
      "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
        "start:dev": "cross-env PORT=8080 nodemon app.js",
        "start:prod": "cross-env PORT=80 node app.js"
      },

Det er vigtigt at nodemon.json og package.json ligger i roden af projektet, på samme niveau - da den ellers ikke klarer at læse nodemon.json.
